# originals-slots — "Slots" verifier

Open-source, MIT reference implementation that lets **anyone** reproduce and verify a settled
**Slots** bet on [maczo Originals](https://verify.maczo.co/originals/) — offline, from the
`serverSeed` / `clientSeed` / `nonce` alone.

This repo is one game in the Originals verifier fleet. The cryptographic core (HMAC-SHA256 → `uint32`
stream, per `libs/provably_fair/fair.py`) lives ONCE in the shared engine
[`@maczo/originals-verify`](https://github.com/maczo-co/originals-verify); this repo adds only the pure
`Slots` **resolver** and its published odds.

- **Bias class:** `modulo` — reduces a uint32 by modulo, carrying a documented-negligible bias (< ~1e-5).
- **Recipe:** 15 cells drawn by weight (index 0 rarest-last); each of 3 rows pays its left-aligned run≥3 of matching symbols via runMultiplierE8[symbol][run]; rows add up.

## Files

- `resolve.js` — the pure resolver: `resolve(uints, params, paytable, { rtpE8, betMinor })` → `{ multiplierE8, win, payoutMinor, outcome }`. Mirrors `libs/game_math/slots.py`.
- `paytable.json` — the published odds + the exact `e8` tables the resolver reads (computed by the server at RTP 99%).
- `test-vectors.json` — known-answer vectors generated by the authoritative Python.
- `verify.js` — the self-test CLI.

## Verify a settled bet yourself

```
npm install
npm test        # reproduces every known-answer vector (JS ⇄ Python)
```

To check one real bet, deep-link into the shared verifier with your bet's
`server` (revealed seed), `commit`, `client`, `nonce` and `params` — see the docs link above.
